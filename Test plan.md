<a name="introduction"/>

## 1. Введение

Данный документ описывает план тестирования десктопного приложения [KitchenMate](https://github.com/tokisero/KitchenMate) для Windows.  
Документ предназначен для участников команды разработки и тестирования.  
Цель тестирования — проверить, соответствует ли поведение приложения требованиям, описанным в [SRS](https://github.com/tokisero/KitchenMate/blob/main/docs/Requirements/SRS.md).

---

<a name="items"/>

## 2. Объект тестирования

К объектам тестирования относятся основные функции десктопного приложения KitchenMate для Windows:

* Управление виртуальным складом продуктов (pantry);  
* Поиск рецептов на основе имеющихся ингредиентов;  
* Генерация списков покупок;  
* Сохранение и просмотр избранных рецептов;  
* Ручное добавление пользовательских рецептов;  
* Отображение интерфейса навигации (главное окно, склад, рецепты, списки).

---

<a name="quality"/>

## 3. Атрибуты качества

1. **Функциональность:**  
   - Все заявленные функции должны выполняться корректно;  
   - Все формы должны проходить валидацию перед отправкой данных.  

2. **Удобство использования:**  
   - Интерфейс должен быть понятен и интуитивен;  
   - Навигация должна адаптироваться под десктопные разрешения экранов;  
   - Все кнопки и поля имеют осмысленные подписи.  

3. **Производительность:**  
   - Время загрузки окон не должно превышать 3 секунд.  

4. **Безопасность:**  
   - Интеграция с API Spoonacular должна быть защищена (без раскрытия ключей);  
   - Данные склада и списков сохраняются в локальном хранилище.

---

<a name="risk"/>

## 4. Риски

Возможные риски при тестировании и эксплуатации:

* Отсутствие соединения с сервером API Spoonacular;  
* Ошибки поиска и генерации списков;  
* Потеря данных при незавершённом добавлении элемента;  
* Некорректная работа интерфейса на разных разрешениях экранов;  
* Несовместимость с отдельными версиями Windows.

<a name="features"/>

## 5. Аспекты тестирования

Планируется тестирование следующих аспектов приложения:

### Управление складом продуктов
- Проверка формы добавления элемента в склад;  
- Проверка обязательных полей (название, количество);  
- Просмотр и редактирование списка склада.

### Поиск рецептов
- Поиск по одному и нескольким ингредиентам из склада;  
- Проверка корректности результатов поиска через API;  
- Обработка пустого склада.

### Генерация списков покупок
- Создание списка из выбранного рецепта;  
- Добавление/удаление элементов в список;  
- Отображение недостающих ингредиентов.

### Работа с избранным и пользовательскими рецептами
- Сохранение рецепта в избранное;  
- Ручное добавление нового рецепта;  
- Просмотр и удаление из списка избранного.

---

<a name="approach"/>

## 6. Подходы к тестированию

Используется **ручное функциональное тестирование**.  
Проверка проводится на **Windows 10/11**.  
Инструменты:  
- Windows Task Manager для мониторинга производительности;  
- Postman для проверки запросов API Spoonacular;  
- Локальное хранилище для симуляции данных.

---

<a name="pass"/>

## 7. Представление результатов

Результаты тестирования оформлены в отдельном документе [Test Results](Test%20Results.md).  
Каждый тестовый сценарий содержит следующие поля:
| ID | Назначение / Описание | Сценарий / Инструкции | Ожидаемый результат | Фактический результат | Оценка (Pass/Fail) |

<a name="conclusion"/>

## 8. Выводы

Данный тестовый план охватывает все основные функции приложения KitchenMate.  
Успешное прохождение тестов подтверждает корректность работы системы в штатных условиях.  
Возможные дополнительные виды тестирования (нагрузочные, кросс-версионные, UX-тесты) могут быть добавлены на следующих этапах разработки.
